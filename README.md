# Ozon Tracking Bot

Telegram-бот: следит за заказами на `tracking.ozon.ru` и шлёт уведомления при смене статуса. Всё управление — треки и cookies — через чат, без перезапуска контейнера.

## Настройка

1. `cp .env.example .env` — заполнить:
   - `BOT_TOKEN` — токен от @BotFather
   - `CHAT_ID` — id чата, которому разрешено управление и куда идут алёрты (через @userinfobot)
   - `TRACK_URLS` *(опционально)* — ссылки через запятую для первичного засева `data/tracks.json`. Дальше треки управляются через ТГ.
   - `DEBUG_DUMP=1` *(опционально)* — включает дамп HTML/текста страницы в `data/debug/` при ошибках.

2. **Куки от Ozon** (обязательно — у Ozon агрессивный antibot):
   - Откройте `https://www.ozon.ru/` в обычном браузере.
   - Поставьте расширение **Cookie-Editor** (Chrome/Firefox).
   - На странице ozon.ru → Cookie-Editor → **Export** → **JSON**.
   - Пришлите боту JSON-файлом или текстом — он сохранит его в `data/cookies.json`.
   - Альтернатива: положить файл руками в `./data/cookies.json` до запуска.
   - Поддерживается также формат Playwright `storage_state.json`.

3. `docker compose up -d --build`

4. В Telegram напишите боту `/start` — придёт сообщение со списком заказов и кнопкой «🔄 Обновить».

## Использование

- `/start` — закрепляемое сообщение со статусами и кнопкой «🔄 Обновить».
- **Добавить трек:** пришлите ссылку `https://tracking.ozon.ru/?track=...` или просто id трека (`XXXXXXXX-XXXX-X`). Бот сразу подтянет статус и обновит закреп. Можно несколько за раз.
- **Удалить трек:** в блоке заказа есть скрытая под спойлером команда `/del_<uid>` — тапнул, бот удалил.
- **Обновить вручную:** кнопка «🔄 Обновить» (троттлинг 10 сек на чат, общий лок на параллельные запуски).
- **Авто-проверка:** раз в 30–59 мин (рандом). Время последнего и ожидаемого следующего показано в закрепе.
- **Алёрты:** при изменении статуса прилетает отдельное сообщение со старым→новым этапом, описанием и датой. У всех ответов есть кнопка «Понятно», которая удаляет сообщение бота. Сообщения пользователя (команды/ссылки/куки) бот удаляет сам.

Управление (`/del_*`, приём ссылок и cookies) доступно только из чата с `CHAT_ID` из `.env`.

## Ограничения нагрузки на Ozon

Чтобы не дёргать сервисы Ozon чаще, чем нужно, в код встроены лимиты:

- **Авто-проверка:** интервал случайный в диапазоне **30–59 мин** (`POLL_MIN_SEC` / `POLL_MAX_SEC`). Рандом размазывает запросы во времени — нет ровных «пиков».
- **Параллелизм:** одновременно крутится не более **5** браузерных сессий (`CONCURRENCY`, верхняя граница жёстко зажата в коде).
- **Ручной троттлинг:** кнопка «🔄 Обновить» блокируется на **10 сек** на чат (`REFRESH_COOLDOWN_SEC`). Чаще нажать нельзя — приходит «Слишком часто! Подождите».
- **Глобальный лок:** одновременно работает только один цикл обновления — несколько пользователей не могут запустить параллельные fetch'и. Пока идёт обновление, кнопка во всех чатах превращается в «⏳ Обновление...».
- **Дедупликация треков:** одну и ту же ссылку повторно добавить нельзя — ответ «Уже отслеживается».
- **Один запрос на трек за цикл:** перехватываем ровно один API-ответ `/p-api/ozon-track-bff/tracking/<id>` через Playwright и сразу закрываем браузер; никаких лишних навигаций.
- **Засев state при добавлении:** новый трек один раз фетчится сразу, после чего ближайший авто-цикл уже не считает его «изменившимся» — нет повторных алёртов.

Переопределить лимиты можно через env:

```
POLL_MIN_SEC=1800   # минимальный интервал авто-пула, сек
POLL_MAX_SEC=3540   # максимальный интервал авто-пула, сек
CONCURRENCY=5       # параллельных браузеров (1..5)
```

Не уменьшайте интервал агрессивно — это плохо для Ozon и быстро спалит ваши cookies.

## Файлы в `./data/`

- `cookies.json` — куки Ozon. Обновляются присылкой в ТГ.
- `tracks.json` — единый файл `{uid: {url, state}}`: список треков и снимок последнего статуса.
- `pinned_message.json` — id закреплённых сообщений по чатам.
- `logs/bot.log` — логи с ротацией (5 МБ × 3 ≈ 15 МБ).
- `debug/<track>.html|.txt` — дампы страницы (только если `DEBUG_DUMP=1`).

Всё содержимое `data/` создаётся автоматически.

## Куки протухли

Если в статусе появилось `antibot (обнови cookies.json)` — заново экспортируйте JSON из Cookie-Editor и пришлите боту файлом или текстом. Перезапуск не нужен.
